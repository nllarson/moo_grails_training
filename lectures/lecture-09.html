Lecture 09

Groovy Server Pages (gsp)

  • .gsp files work very similar to .jsp files, with the enhancement that you can:
  • use Groovy expressions i.e. ${} for text substitutions
  • g: tags are Grails aware of default Grails conventions and MVC architecture
  • Plugins like the Spring Security Plugin will often add tag libraries automatically

<div id="main-nav">
  <sec:ifLoggedIn>Welcome <sec:username/>
    <g:link controller="logout">logout</g:link>
  </sec:ifLoggedIn>
</div>

Layout Conventions

  • Place images in web-app/images folder
  • Global Template is in views/layouts
  • templates are implemented using Sitemesh
  • default template is views/layouts/main.gsp

Layout Conventions - Continued

  • CSS files are stored in web-app/css
  • the default CSS styling is in webapp/css/main.css
  • include resources using the resource method
  • Templates can used to capture reusable bits of view code
  • Custom taglibs are easy to create and easy to; way easier than standard JSP taglibs

Layout Example


-- grails-app/views/layouts/main.gsp
<html>
  <head>
    <title><g:layoutTitle default="Timesheet App"/></title>
    <g:layoutHead/>
  </head>
  <body>
    <div class="menu">
      <ul>
        <li>Home</li>
        <li>Current Timesheet</li>
        <li>Logout</li>
      </ul>
    </div>
    <div class="body">
      <g:layoutBody/>
    </div>
  </body>
</html>
